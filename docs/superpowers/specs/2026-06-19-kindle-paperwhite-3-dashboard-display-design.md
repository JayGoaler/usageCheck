# Kindle Paperwhite 3 仪表盘显示尺寸规范

**日期:** 2026-06-19  
**状态:** 已批准  
**目标设备:** Kindle Paperwhite 3（第 7 代，2015，型号 DP75SDI）

---

## 1. 目标

定义 AI Usage Check 仪表盘在 Kindle Paperwhite 3（PW3）上的设计基线，包括屏幕尺寸、网页视口处理方式、布局尺寸、字号、触控区域和电子墨水屏限制。

本规范只约束 Kindle 展示层，不改变电脑端的 FastAPI、Collector 和 SQLite 架构。

---

## 2. 硬件屏幕信息

| 项目 | PW3 参数 |
|------|---------|
| 发布时间 | 2015-06-30 |
| 型号 | DP75SDI |
| 屏幕类型 | 6 英寸 E Ink Carta HD |
| 灰阶 | 16 级灰度 |
| 像素密度 | 300 PPI |
| 竖屏物理分辨率 | **1072 × 1448 px** |
| 横屏物理分辨率 | 1448 × 1072 px |
| 竖屏宽高比 | 0.7403 |
| 横屏宽高比 | 1.3507 |
| 推算有效显示宽度 | 约 90.7 mm |
| 推算有效显示高度 | 约 122.5 mm |
| 机身尺寸 | 约 117 × 169 × 9.1 mm |
| 输入方式 | 电容触摸屏 |

有效显示尺寸由 6 英寸对角线、1072×1448 分辨率按比例推算：

```text
屏幕宽度约 3.570 英寸 = 90.7 mm
屏幕高度约 4.822 英寸 = 122.5 mm
像素对角线约 1801.6 px
推算密度约 300.3 PPI
```

旧设计文档中的 600×800 是早期普通 Kindle 的分辨率级别，不是 PW3 的物理分辨率。

---

## 3. 物理像素不等于 CSS 视口

`1072×1448` 只能确定面板硬件像素，不能直接当作网页的 CSS 布局尺寸。Kindle 浏览器可能受以下因素影响：

- 固件版本。
- 浏览器顶部/底部工具栏。
- 页面缩放状态。
- 浏览器对高 PPI 屏幕的 CSS 像素映射。
- `viewport` meta 标签支持情况。
- 竖屏或横屏方向。

因此，不在代码中假定：

```text
window.innerWidth == 1072
window.innerHeight == 1448
devicePixelRatio == 1
```

页面必须保留：

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

但仍以百分比宽度、单列布局和媒体查询处理实际视口。

---

## 4. 实机视口校准

开发阶段增加一个临时诊断页面或调试区域，在 PW3 浏览器中显示：

```javascript
{
  screenWidth: screen.width,
  screenHeight: screen.height,
  innerWidth: window.innerWidth,
  innerHeight: window.innerHeight,
  clientWidth: document.documentElement.clientWidth,
  clientHeight: document.documentElement.clientHeight,
  devicePixelRatio: window.devicePixelRatio || 1,
  userAgent: navigator.userAgent
}
```

需要分别记录：

1. 竖屏、浏览器工具栏显示。
2. 竖屏、页面进入最大可用显示状态。
3. 横屏状态（如果仪表盘需要支持）。
4. 页面缩放恢复默认后的结果。

在取得实机数据前，验收以“无横向滚动、文字清晰、按钮可触摸、主要状态可在少量滚动内读取”为准，不以某个未经测量的 CSS 视口值为准。

## 5. 布局方案比较

### 5.1 单列纵向卡片（采用）

```text
┌────────────────────┐
│ AI 用量       刷新 │
├────────────────────┤
│ Codex              │
│ 5 小时  86%        │
│ 7 天    98%        │
├────────────────────┤
│ DeepSeek           │
│ 余额 / 月度用量     │
├────────────────────┤
│ 网络流量            │
│ 剩余 / 到期时间     │
└────────────────────┘
```

优点：

- 适合约 90.7 mm 的实际屏幕宽度。
- 字号和触控区域可以保持足够大。
- 不依赖 CSS Grid。
- 内容顺序清晰，电子墨水屏滚动次数可控。

### 5.2 双列卡片（不采用）

虽然面板有 1072 个横向像素，但实际屏幕只有约 90.7 mm 宽。双列会让每张卡片的物理宽度不足，导致文字缩小、换行和误触。

### 5.3 固定 600×800 画布（不采用）

该方案混淆了旧 Kindle 分辨率和 PW3 CSS 视口，可能造成放大、留白或横向滚动。

## 6. 推荐展示尺寸

以下均为 CSS 布局值，需要通过 PW3 实机诊断结果最终校准。

### 6.1 页面

| 元素 | 推荐值 |
|------|--------|
| 页面宽度 | `width: 100%` |
| 最大宽度 | 不设置固定 `600px` 上限 |
| 页面外边距 | `0` |
| 左右安全留白 | 窄视口 12px；较宽视口 20–24px |
| 顶部留白 | 12–16px |
| 卡片间距 | 12–16px |
| 底部留白 | 至少 24px |
| 横向溢出 | 禁止，`overflow-x: hidden` 仅作防御，不代替正确布局 |

实现时优先使用旧浏览器可靠的媒体查询：

```css
.page {
  width: auto;
  margin: 0;
  padding: 12px;
}

@media (min-width: 800px) {
  .page {
    padding: 20px 24px;
  }
}
```

避免依赖 `clamp()`、CSS Grid 和复杂容器查询。

### 6.2 顶部区域

| 元素 | 推荐高度/尺寸 |
|------|--------------|
| 页面标题 | 30–36px |
| 更新时间 | 20–24px |
| 顶部区域最小高度 | 72–88px |
| 刷新按钮高度 | 56–64px |
| 刷新按钮最小宽度 | 112px |

### 6.3 数据卡片

| 元素 | 推荐值 |
|------|--------|
| 卡片宽度 | 100% |
| 卡片内边距 | 18–24px |
| 卡片最小高度 | 230–280px |
| 卡片标题 | 26–32px |
| 普通标签 | 20–24px |
| 普通数值 | 28–36px |
| 核心百分比/余额 | 48–64px |
| 行高 | 1.35–1.5 |
| 分隔线 | 2px 实线 |
| 卡片边框 | 2–3px 实线 |
| 圆角 | 0–6px，允许直接使用直角 |

三张主要卡片按每张约 250px、卡片间距 14px 估算，内容高度约 778px；加上标题区后，主要信息可在一屏到约一屏半内读取，具体取决于实际 CSS 视口高度。

### 6.4 触控区域

WCAG 2.2 的最低目标尺寸为 24×24 CSS px。考虑 PW3 触控精度、屏幕刷新延迟和未知 CSS 像素映射，本项目采用更保守的尺寸：

| 控件 | 项目要求 |
|------|---------|
| 普通按钮最小高度 | 56px |
| 主要按钮建议高度 | 64px |
| 图标按钮最小区域 | 56×56px |
| 相邻按钮间距 | 至少 12px |
| 可点击文本 | 不作为唯一主要操作入口 |

## 7. 信息密度和优先级

PW3 首屏信息按以下优先级排列：

1. Codex 5 小时剩余百分比和重置时间。
2. Codex 7 天剩余百分比和重置时间。
3. DeepSeek 余额和当月费用。
4. 网络剩余流量和到期时间。
5. 最近成功更新时间。
6. 错误和数据过期状态。

首屏不展示：

- 多点折线图。
- 动态图表。
- 长历史列表。
- 大段帮助文字。
- 装饰性图片。

历史数据通过独立页面或折叠区域访问。

## 8. 电子墨水屏视觉规则

PW3 是 16 级灰度电子墨水屏，应采用高对比、低动态设计：

- 默认白底黑字。
- 关键边框使用纯黑或深灰。
- 不依赖颜色表达成功、警告或错误。
- 状态必须同时使用文字、符号或边框样式。
- 不使用渐变、半透明玻璃效果、细阴影或背景纹理。
- 不使用连续动画、进度条动画、闪烁或自动轮播。
- 进度条使用黑色填充、白色空白和明显边框。
- 数据变化时只更新必要 DOM，减少重绘和残影。
- 自动轮询建议保持 60 秒，不因轮询而强制整页刷新。
- 提供手动刷新按钮，但刷新期间不要显示旋转动画。

建议状态表达：

```text
正常： [正常] 86%
较低： [注意] 18%
异常： [错误] 无法获取
过期： [旧数据] 更新于 12:30
```

## 9. 断点和方向

### 9.1 竖屏

竖屏是 PW3 的主要模式：

- 单列。
- 卡片宽度 100%。
- 页面不出现横向滚动。
- 主要操作位于页面顶部。

### 9.2 横屏

横屏仍保持单列，不因为宽度增加自动切换成三列。可以在实机确认空间足够后考虑两列，但不属于首版要求。

### 9.3 推荐断点

```css
/* 默认：Kindle 和窄屏设备 */
@media (min-width: 800px) {
  /* 只增加留白和字号，不改变为多列 */
}

@media (min-width: 1200px) {
  /* 桌面浏览器可以启用三列 */
}
```

断点基于实际 CSS 视口，而不是面板硬件分辨率。

## 10. 浏览器兼容约束

- 使用语义化 HTML 和普通块级布局。
- JavaScript 使用保守语法，避免要求现代打包运行时。
- 为 `fetch`、SSE 等能力提供失败处理或轮询回退。
- 不依赖 CSS Grid、复杂 Flexbox 行为、CSS 自定义属性或新选择器才能读取核心数据。
- 即使 JavaScript 部分失败，也应显示页面框架和明确错误信息。
- 所有外部字体和图标均取消；使用系统字体和文本符号。
- 页面资源尽量保持单文件或少量本地静态文件，减少 Wi-Fi 请求。

---

## 11. 验收标准

在真实 PW3 上验证：

- 记录 `screen.*`、`inner*`、`client*`、`devicePixelRatio` 和 User-Agent。
- 竖屏默认缩放下无横向滚动。
- 标题、数值、重置时间无需缩放即可辨认。
- 所有按钮可稳定触发，且相邻按钮不会误触。
- Codex 两个额度窗口在首张卡片完整显示。
- 三类核心状态在一屏到一屏半内读取。
- 页面首次加载后没有持续动画。
- 60 秒轮询不会引起整页闪烁。
- 灰阶环境下不依赖颜色也能区分状态。
- 浏览器工具栏出现时，页面仍可正常滚动和操作。

实机视口数据采集后，应将实际值追加到本文件，而不是替换硬件分辨率数据。
