# Kindle 响应式仪表盘修复设计

**日期：** 2026-06-19  
**状态：** 待用户审核  
**目标设备：** Kindle Paperwhite 3（第 7 代，PW3）  
**参考视觉：** `dashboard-test.png`

## 1. 目标

修复 Kindle 通过局域网访问 `http://<主机 IP>:8080/` 时的三个问题：

1. 页面不能适配 Kindle 横竖屏及可用视口。
2. 页面框架可以打开，但动态数据不显示。
3. 当前深色背景在电子墨水屏上对比度不足。

本次只调整仪表盘前端及必要的兼容性测试，不改变 Collector、SQLite 或 `/api/dashboard` 的数据结构。

## 2. 已确认根因与约束

- FastAPI 已监听 `0.0.0.0:8080`。
- 本机访问 `/api/dashboard` 能返回完整的 DeepSeek、Codex 和网络流量数据。
- 前端使用 `const`、`let`、`async/await` 和 `fetch`。PW3 的旧版实验浏览器可能在解析或执行这些语法/API 时失败，造成整段脚本不运行。
- 当前错误只写入 `console.error`，Kindle 页面没有可见反馈。
- 当前页面使用深色背景、低对比辅助文字、Grid 和动画进度条，不适合 16 级灰度电子墨水屏。
- PW3 的物理分辨率不能直接等同于 CSS 视口，布局必须依据实际可用视口自适应。

## 3. 采用方案

采用“竖屏单列、横屏 1+2”的响应式布局。

### 3.1 竖屏

从上到下显示：

1. 顶部标题、全屏按钮、刷新按钮。
2. Codex 卡片。
3. DeepSeek 卡片。
4. 网络流量卡片。
5. 最近更新时间或错误信息。

三张卡片均占满可用宽度。通过紧凑间距和明确字号，尽量在全屏状态下一屏显示；若实机 CSS 视口不足，允许最小程度的纵向滚动，但禁止横向滚动。

### 3.2 横屏

- Codex 卡片占据第一整行。
- DeepSeek 与网络流量卡片在第二行左右并排。
- 不使用三列，避免文字、重置时间和触控按钮被压缩。

横屏双列使用兼容旧浏览器的 `inline-block` 或可靠的基础 Flexbox，并提供块级单列回退；核心内容不依赖 CSS Grid。

## 4. Codex 卡片

Codex 的两个额度窗口始终上下排列，横竖屏行为一致：

### 4.1 5 小时额度

- 标题“5 小时额度”。
- 剩余百分比。
- 独立柱状进度条。
- 明确的重置时间。

### 4.2 7 天额度

- 标题“7 天额度”。
- 剩余百分比。
- 独立柱状进度条。
- 明确的重置时间。

两个进度条使用深色实心填充、白色空余区域和清晰边框。百分比文字是主要信息，颜色只作辅助。

## 5. 视觉设计

整体参考 `dashboard-test.png` 的浅色仪表盘方向，并针对电子墨水屏简化：

- 页面使用白色或近白色背景。
- 正文和关键数值使用黑色或深灰。
- 卡片使用浅色底、清晰边框和小圆角。
- DeepSeek、网络流量和 Codex 可使用非常浅的蓝、橙、紫色底作为桌面浏览器辅助色，但灰阶下仍须可区分。
- 不使用渐变、玻璃效果、细阴影或依赖颜色表达状态。
- 不使用连续动画、进度条过渡、闪烁和自动轮播。
- 保留足够大的标题、数值和触控按钮。
- 不加载外部字体或图标资源。

## 6. Kindle JavaScript 兼容策略

前端脚本改为保守的 ES5 风格：

- 使用 `var` 和普通函数。
- 不使用 `async/await`、箭头函数、模板字符串、可选链或 Promise 作为核心依赖。
- 使用 `XMLHttpRequest` 请求 `/api/dashboard`。
- JSON 解析失败、HTTP 非 2xx、超时或网络错误时，在页面顶部显示可见错误。
- 60 秒轮询保留，但每次只更新数据节点，不刷新整个页面。
- 手动刷新按钮复用同一请求函数。

接口使用同源相对路径 `/api/dashboard`，因此从 `http://<主机 IP>:8080/` 打开时仍请求同一主机，不引入 CORS。

## 7. 全屏行为

页面提供“全屏”按钮，按以下顺序尝试：

1. `document.documentElement.requestFullscreen()`。
2. WebKit 前缀全屏 API。
3. 其他浏览器前缀 API（若存在）。

如果 PW3 浏览器不支持网页全屏 API，页面显示固定文字提示：

> 当前浏览器不支持网页全屏，请通过 Kindle 浏览器菜单进入全屏或隐藏工具栏。

网页不能保证强制隐藏 Kindle 浏览器自身工具栏，因此“不支持时清晰提示”属于验收行为，而不是静默失败。

## 8. 视口适配

- 保留 `width=device-width, initial-scale=1.0`。
- 页面宽度使用百分比，不设置固定画布宽度。
- 使用 `orientation` 和基础宽度媒体查询识别横屏。
- 使用 `min-height: 100%`，并根据可用高度压缩标题、间距和卡片内边距。
- 禁止横向溢出。
- 不假设 `window.innerWidth` 等于 1072 或 1448。

增加一个默认隐藏的诊断区域，可通过查询参数 `?debug=1` 显示：

- `screen.width` / `screen.height`
- `window.innerWidth` / `window.innerHeight`
- `documentElement.clientWidth` / `clientHeight`
- `devicePixelRatio`
- User-Agent

这些数据用于实机校准，不影响普通仪表盘。

## 9. 错误与空数据状态

- 页面加载后立即显示“正在读取数据”。
- 请求失败时显示连接错误和最近一次成功更新时间。
- 单个数据源缺失时，只在对应卡片显示“暂无数据”，其他卡片继续渲染。
- Codex 不可用时显示明确状态，不伪造 100%。
- 保留旧数据时标记“旧数据”，避免把缓存值误认为实时值。

## 10. 测试与验收

### 10.1 自动检查

- `/api/dashboard` 返回的现有字段可被兼容脚本正确渲染。
- 页面核心脚本不包含 Kindle 风险语法。
- 模拟接口成功、HTTP 错误、无效 JSON、超时和单数据源缺失。
- 横竖屏样式下均无横向溢出。

### 10.2 桌面浏览器验证

- 使用 PW3 竖屏和横屏比例进行视口截图。
- 验证 Codex 两条额度柱始终上下排列。
- 验证横屏为 Codex 整行、下方两卡并排。
- 验证全屏按钮成功或显示回退提示。

### 10.3 PW3 实机验收

- 通过局域网 IP 和端口打开页面后能读取真实数据。
- 竖屏与横屏切换后布局自动调整，无需重新加载。
- 全屏模式下主要信息尽量在一屏可见。
- 页面无横向滚动。
- 灰阶下标题、数值、边框和进度条清晰。
- 两个 Codex 额度窗口均有独立百分比、柱状条和重置时间。
- 60 秒轮询不引起整页闪烁。
- 记录 `?debug=1` 的实机视口数据并追加到既有 PW3 尺寸规范。
